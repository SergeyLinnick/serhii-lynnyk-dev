# Database Package

The `@workspace/db` package provides the database layer using Drizzle ORM with Neon Postgres (`@neondatabase/serverless`). It exports the Drizzle client, all table schemas, and migration utilities.

## Drizzle Client

The main export is the `db` client, guarded by `server-only` to prevent accidental client-side imports. The client uses lazy initialization via a Proxy so the module can be imported during build without requiring `DATABASE_URL`:

```ts
// src/index.ts
import "server-only";

import { neon } from "@neondatabase/serverless";
import { drizzle, type NeonHttpDatabase } from "drizzle-orm/neon-http";
import * as schema from "./schema";

function createDb() {
	const sql = neon(process.env.DATABASE_URL!);
	return drizzle(sql, { schema });
}

let instance: NeonHttpDatabase<typeof schema> | undefined;
export const db = new Proxy({} as NeonHttpDatabase<typeof schema>, {
	get(_, prop) {
		instance ??= createDb();
		const value = Reflect.get(instance, prop);
		return typeof value === "function" ? value.bind(instance) : value;
	},
});
```

## Schema Conventions

- **Table names:** singular (`task`, `user`, not `tasks`, `users`)
- **Column names:** snake_case in the database (`created_at`, `user_id`)
- **Primary keys:** UUID with `defaultRandom()`
- **Timestamps:** `created_at` and `updated_at` with `defaultNow()`, both `notNull()`
- **Foreign keys:** always define `onDelete` behavior (usually `cascade`)
- **Enums:** use `text` with `enum` option for small enum sets

### Example Table

```ts
import { pgTable, text, timestamp, uuid } from "drizzle-orm/pg-core";
import { user } from "./auth";

export const task = pgTable("task", {
	id: uuid("id").defaultRandom().primaryKey(),
	title: text("title").notNull(),
	description: text("description"),
	status: text("status", {
		enum: ["todo", "in_progress", "done", "cancelled"],
	})
		.notNull()
		.default("todo"),
	userId: text("user_id")
		.notNull()
		.references(() => user.id, { onDelete: "cascade" }),
	createdAt: timestamp("created_at").defaultNow().notNull(),
	updatedAt: timestamp("updated_at").defaultNow().notNull(),
});
```

## Adding a New Table

1. Create `packages/db/src/schema/<domain>.ts` with the table definition.
2. Export from `packages/db/src/schema/index.ts`.
3. Run `pnpm --filter @workspace/db db:generate` to generate a migration.
4. Run `pnpm --filter @workspace/db db:migrate` (or `db:push` for dev) to apply.
5. Create corresponding schemas in `@workspace/models` using `drizzle-zod`.

## Migration Workflow

```bash
# Generate migration from schema changes
pnpm --filter @workspace/db db:generate

# Apply migrations to database
pnpm --filter @workspace/db db:migrate

# Push schema directly (dev only, no migration file)
pnpm --filter @workspace/db db:push

# Open Drizzle Studio (visual DB browser)
pnpm --filter @workspace/db db:studio
```

## Auth Tables

The auth schema (`src/schema/auth.ts`) contains tables required by better-auth:

- `user` ‚Äî user accounts
- `session` ‚Äî active sessions
- `account` ‚Äî OAuth/credential accounts
- `verification` ‚Äî email verification tokens

These tables are generated using the better-auth CLI (`npx @better-auth/cli generate`) and should not be hand-edited. If the auth schema needs updating, re-run the CLI.

## Environment Variables

| Variable                | Description                               |
| ----------------------- | ----------------------------------------- |
| `DATABASE_URL`          | Neon Postgres connection string (pooled)  |
| `DATABASE_URL_UNPOOLED` | Direct connection string (for migrations) |

These are declared in `turbo.json` under `globalEnv`.
